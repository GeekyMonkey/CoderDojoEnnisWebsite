Godot


Godot is a game development tool that helps people create 2D and 3D games. It’s great for beginners and experienced developers. Godot is open-source, meaning it’s free to use and constantly improved by a community of developers.

With Godot, you can start making games without needing to know complex coding right away. It uses an easy-to-understand system of Nodes and Scenes to build projects. You can make anything from simple animations to full games. Godot supports different programming languages like GDScript, C#, and C++, so you can choose what works best for you.

Godot is available for Windows, Mac Os and Linux. The source code is made available under a license that allows modifications for non-commercial uses.

You can download GODOT for free from the Official Website or try the website version Version 4.3 which adds loads of new features.

There is a good introduction document that you can read here: Scratchy Godot . And lots of introductory videos on YouTube, like this one.

Scratch


Scratch is an early computer language learning environment enabling young beginners to get results without having to learn syntactically correct writing first. Created by the MIT Media Lab, it is intended to motivate for further learning through playfully experimenting and creating projects, such as interactive animations etc.

The name Scratch derived from the turntablism’s technique of scratching and stresses the similarity of the process of programming to the process of mixing/scratching instead of its similarity to the processes of writing (i.e. scripting).

Scratch is available for Windows, Mac OS X, and Linux. The source code is made available under a license that allows modifications for non-commercial uses.

You can download SCRATCH for free from the Official Website or try the Version 2.0 which adds loads of new features. There you can also setup an online account to share your projects with the world. When you upload your projects from Scratch, others will be able to play your games from the Scratch website.

There is a good introduction document that you can download and read. And lots of introductory videos on YouTube, like this one.

Gravity and Platforms

As a first step torwards building a Super Mario type game, we talked about how gravity can be simulated with 2 variables. One for the gravity strength (constant), and one for the vertical speed of the player. We check if the player is touching the ground. If he is, then we set his vertical speed to 0 and don’t change his position, otherwise we increase his vertical speed by the gravity strength and then updade his position by his speed. Jumping was performed by just giving him a large upward vertical speed which would be taken away by gravity.


You can download the project here.

Broadcast

Project demonstrated on 24/1/15.

This is a simple project demonstrating how to use the ‘broadcast message’ feature. You use the mouse to move the first sprite towards the other sprite on the screen. When they touch, another sprite appears. You move this sprite to touch the last sprite that appeared and then another one appears – and so on.

When two sprites touch (‘meet’) a message is broadcast that the next sprite receives. Receiving this message is what tells that latter sprite to reveal itself.

You can see the project here.

Maze with Dots

Project demonstrated on 7/2/15.
This is the maze we demonstrated in the beginner class. It uses multiple sprites to act as the little red dots that get ‘eaten’ by the ball. This allows us to delete the sprite as soon as it’s touched so that the score only goes up by increments of 1.
When the ball gets to the end it uses the Broadcast feature to says it’s completed. Before displaying a finish screen, the score will be checked to see if all the red dots were eaten. Depending on the final score, one of two final screen sprites will be displayed.
You can see the project here.

Number Guessing Game

Project demonstrated on 21/3/15.
This is the number guessing game where a monkey sprite comes up with a number between 1 and 100 and asks the user to guess it. The monkey tells the user if the guess is too low or two high and the user gets to guess again.
This project demonstrates how to use variables and respond to data that’s entered on request.
